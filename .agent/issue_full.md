## Issue 24: Cloudinaryストレージ設定

- **概要**: Active Storage と Cloudinary を連携させ、本番環境でハーブ画像のアップロード・表示を動作させる
- **背景**: Issue 5 で gem は導入済みだが、本番環境で画像保存に失敗している。`config/initializers/content_security_policy.rb` が全コメントアウト状態のため、Cloudinary（`res.cloudinary.com`）からの画像読み込みがブラウザにブロックされている可能性がある
- **タスク**:
  - [ ] `config/initializers/content_security_policy.rb` のコメントアウトを解除し、CSP を有効化する
  - [ ] `policy.img_src` に `'https://res.cloudinary.com'` が含まれることを確認する
  - [ ] Docker を再起動して設定を反映させる
  - [ ] 本番環境の `.env` に `CLOUDINARY_CLOUD_NAME` / `CLOUDINARY_API_KEY` / `CLOUDINARY_API_SECRET` が正しく設定されているか確認する
- **完了目安**: 本番環境でハーブ画像が Cloudinary にアップロード・表示される
